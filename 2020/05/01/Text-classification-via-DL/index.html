Convolutional Neural Networks for Sentence Classification

  1. 输入：词向量$x\in R^{300}$

    • CBoW预训练

    • 不在BoW预训练中的词随机初始化

  2. 网络的结构：

    1. CNN-rand:
      1. $c_i=f( \textbf{w} *\textbf{x}_{i:i+h-1}+b)$,$\textbf{w}$是filter的权重，$h$是窗口大小.
      2. 一个sentence经过一个filter后得到一个feature map $\textbf{c}=[c_1,c_2,…,c_{n-h+1}]$.
      3. 一个feature map经过max pool得到 $\hat{c}=max(\textbf{c})$.
      4. 一个sentence经过$m$个filter得到$\textbf{z}=[\hat{c}_1,\hat{c}_2,…,\hat{c}_m]$.
      5. 最后$z$经过全连接层得到$y=\textbf{w}*\textbf{z}+b$.（原文如此，这里的$\textbf{w}$是全连接层一个neuron的权重，区别于1.中filter的权重$\textbf{w}$）
      6. 所有的词都是随机初始化
    2. CNN-static:同CNN-rand相同的是，词向量输入层（相当于一个embedding）在训练中保持不变，训练时只改变网络的其余部分，不同是使用了预训练的word2vec.
    3. CNN-non-static:同2.不同的是，词向量输入层在训练时也会改变.
    4. CNN-multichannel:使用两个预训练的word2vec的set,每个set看作是一个”channel”,两个channel共用卷积层、全连接层，但是一个channel是static，另一个是non-static.

text classification via RNN

常规的LSTM操作，唯一特殊的地方是，在LSTM的embedding输入进LSTM之前，可以先用一维的卷积层和池化层进行处理，在不影响精度的情况下加速网络训练，这点值得尝试。

word embedding 的若干种方式

  1. 随机：pytorch的nn.embedding即是正态随机初始化的

  2. word2vec:

    1. CBOW:根据上下文来预测当前词，生成副产品词向量
    2. skip-gram:根据当前词来预测上下文，生成副产品词向量
    3. 改进：
      • hierarchical softmax:使用预先根据字典构造的霍夫曼树来替代原先的一维的softmax输出，树上每个叶节点即为字典中的一个词，每个非叶节点即为一个二分类器。简单来讲，反向传播更新梯度时，只更新当前正例所在的叶节点到跟的路径上的梯度，加速网络的训练。
      • negative sampling:根据词频去除一些训练的sample,词频愈高，被down sample的几率愈大。被去除的几率由实现定义的经验公式给出。反向传播时，正样例（当前中心词）是一定会更新的（输入层到隐藏层的矩阵、隐藏层到输出层的矩阵），但是对于负样例，只会负样例的词频（词频越大，被挑选的记录就越大）挑选若干个来更新（只更新隐藏层到输出层的矩阵，输入层到隐藏层的矩阵不会更新，因为输入层one-hot编码）。来加速训练词向量的过程并提高得到的词向量的质量。

  3. GloVe:embedding对于文档的信息利用来说，有两种 :

    • 基于全局的（e.g.,全局矩阵分解，LDA）. 可以较好的利用文本中词的统计信息。

    • 基于局部（上下文的）的（e.g.,CBOW,skip-gram) .可以较好的发掘上下文的语义、词与词间的相似。

      GloVe的基本思想是，融合两种方法。优化目标是，使每两个词的embedding的内积同两两词的共现概率的对数相同。

    • 应该是该数据集比较小的原因，LSTM两层就已经过拟合了，三层跟两层效果相近，一层高出两层、三层许多。

    • 训练和预测的时候有个地方应该可以再发掘下，我现在想到的是把一个句子切分后的phrase和句子本身分开训练，但是考虑在现实中，即使是同一个词在不同的评论中也有可能语义或情感有所差别，所以把phrase跟句子一同训练效果可能更好。
